## M.1 Standards & Institutional Context

### M.1.1 Applicable Standards

| Standard | Edition/Year | Sections Referenced | Status |
|----------|-------------|---------------------|--------|
| Uganda MoWT Road Design Manual | 2010 | Vol. 1 Ch. 2–5 | Current — primary standard for all roads in Uganda |
| UNRA Supplementary Guidelines | Various | Project-specific | UNRA issues supplementary guidelines beyond MoWT for national trunk roads |

Note: The MoWT manual references AASHTO as one of its technical sources (the minimum radius formula, side friction factors, and design vehicle dimensions are derived from AASHTO research). However, AASHTO is not a separate parallel standard used in Uganda — the MoWT manual is the national standard for all roads including UNRA roads.

### M.1.2 Institutional Context

Uganda has a split institutional structure for road management, but the **MoWT Road Design Manual is the national standard for all roads:**

- **UNRA (Uganda National Roads Authority):** Manages national trunk roads (~20,000 km). Uses the **MoWT Road Design Manual** as the base standard, with **UNRA supplementary guidelines** that provide additional requirements specific to national roads (e.g., higher design classes, specific cross-section standards). UNRA projects, particularly those funded by development partners, may reference international standards in the ToR, but MoWT remains the base.
- **District Local Governments:** Manage district roads (~30,000 km) through District Engineering offices. Use the **MoWT Road Design Manual** directly.
- **KCCA (Kampala Capital City Authority):** Manages roads within Kampala. Uses MoWT manual with additional urban design considerations.
- **Municipal/Town Councils:** Manage urban roads outside Kampala. Use MoWT manual.

**Funding source may introduce additional requirements:** Projects funded by development partners (World Bank, AfDB, EU, JICA) may specify additional standards or requirements in the Terms of Reference. These supplement (not replace) the MoWT manual. Always check the ToR for project-specific requirements.

### M.1.3 Standard Philosophy

**Uganda MoWT Road Design Manual (2010):** Design manual approach. Provides tables with recommended values by design class and terrain. The manual states that its contents are "partly guidelines and recommendations to be considered, and partly standards which as a general rule should be adhered to." In special conditions, modifications may be made in consultation with the Engineer-In-Chief, MoWT. There is no formal departures process — deviation from the manual values is at the engineer's professional judgment, documented in the design report.

The manual draws on international sources including AASHTO for its technical foundations (the curve formula, friction factors, and design vehicles are derived from AASHTO research), but it is a standalone Ugandan standard calibrated for Ugandan conditions.

### M.2.3 Classification System

Uganda uses a **two-tier classification system**:

1. **Functional Classification (Classes A–E):** Defines the road's role in the network (Table U.1). Determines the required level of access control and the applicable Design Classes.
2. **Design Classification (Classes Ia, Ib, II, III, A, B, C):** Defines the geometric standard to be applied (Tables U.2–U.4). Governed by design speed and design traffic, within the range permitted by the functional class.

**The mapping is not one-to-one.** A Functional Class C road (Primary) could be designed to Design Class II, III, or A depending on traffic volume and whether it is paved or gravel. The agent must determine both the Functional Class (from network role) and the Design Class (from traffic + surface type + functional class).

**UNRA roads** use the same MoWT classification system but typically apply the higher design classes (Ia, Ib) for national trunk roads. UNRA may have supplementary guidelines that specify additional requirements beyond the MoWT manual — check project-specific documentation.

**Design vehicles for Uganda (from MoWT Vol. 1, Section 5.1):**
| Design Class | Design Vehicle | Description |
|-------------|---------------|-------------|
| Ia | WB-15 | Semi-trailer, 15m wheelbase |
| Ib, II | SU / WB-12 | Single-unit truck or intermediate semi-trailer |
| III, A, B | SU | Single-unit truck |
| C | P / SU | Passenger car (but must accommodate occasional SU) |

## M.3 Country-Specific Reasoning

### M.3.1 Decision Logic (Uganda Branch)

```
Step 2 (Uganda): Determine which standard
  ├── MoWT Road Design Manual is the primary standard for ALL roads in Uganda
  ├── If road is under UNRA
  │     → Primary: Uganda MoWT Road Design Manual (2010)
  │     → Also check: UNRA supplementary guidelines for additional requirements
  ├── If road is under District Local Government
  │     → Primary: Uganda MoWT Road Design Manual (2010)
  ├── If road is under KCCA (Kampala)
  │     → Primary: Uganda MoWT Road Design Manual with urban provisions
  ├── If road is under Municipal/Town Council
  │     → Primary: Uganda MoWT Road Design Manual
  ├── If funded by development partner
  │     → Check: ToR may specify additional requirements beyond MoWT
  └── If low-volume road (< 200 AADT) in rural area
        → MoWT Design Class C or B; also consider TRL ORN 31 as supplementary reference

Step 6a (Uganda — MoWT): Determine Functional Class
  ├── From road's role in the network (see Table U.1):
  │     ├── Links international centres → Class A
  │     ├── Links provincial capitals / nationally important → Class B
  │     ├── Links provincially important centres → Class C
  │     ├── Links locally important centres → Class D
  │     └── Minor link / access to land → Class E
  └── If unclear → Ask user. Default to Class D for district roads.

Step 6b (Uganda — MoWT): Determine Design Class
  ├── From functional class + traffic + surface type:
  │     ├── Class A/B, paved, high traffic (>10,000 pcu/day) → Design Class Ia (4-lane)
  │     ├── Class A/B, paved, moderate traffic (6,000–10,000) → Design Class Ib (2-lane)
  │     ├── Class B/C/D, paved, traffic 4,000–8,000 → Design Class II
  │     ├── Class C/D/E, paved, traffic 2,000–6,000 → Design Class III
  │     ├── Class C/D/E, gravel, traffic 4,000–8,000 → Design Class A
  │     ├── Class D/E, gravel, traffic 2,000–6,000 → Design Class B
  │     └── Class E, gravel, low traffic → Design Class C
  └── Note: Each road must be assessed individually; capacity ranges are guides not rigid thresholds

Step 8 (Uganda): Design vehicle
  ├── Design Class Ia (UNRA national) → WB-15
  ├── Design Class Ib, II → SU or WB-12 (verify truck traffic)
  ├── Design Class III, A, B → SU
  └── Design Class C → P (but check for occasional truck access)

Step 9 (Uganda): Design period
  ├── National Trunk (UNRA / Design Class Ia, Ib) → 20 years
  ├── Design Class II, III → 15 years
  ├── Design Class A, B → 10–15 years
  └── Design Class C → 10 years
```

### M.3.2 Professional Judgment (Uganda-Specific)

**The MoWT manual is the national standard, but project ToRs may introduce additional requirements.** On major projects (particularly donor-funded), the Terms of Reference may reference international standards like AASHTO or ERA for specific elements. These supplement the MoWT manual — they don't replace it. When a ToR references both MoWT and another standard, the engineer should clarify with the client which takes precedence where they conflict.

**Traffic data in Uganda is often limited.** Many district roads have never had a formal traffic count. Short-duration counts (3-day or 7-day) are common but may not capture seasonal variation. For agricultural areas, seasonal peaks can be 2–3× the average (harvest season, planting season). The 5% default growth rate reflects typical secondary road growth in Uganda, but corridors near new developments (oil region in Albertine Graben, industrial parks) may see much higher growth.

**The Community Road class is underserved by formal standards.** Many community roads in Uganda have AADT < 20. The MoWT manual provides Design Class C (gravel, single lane) as the lowest class, but engineering judgment plays a larger role at this level. TRL ORN 31 can be a useful supplementary reference for very low-volume roads.

**Overloading is endemic on Ugandan roads.** Heavy goods vehicles frequently exceed legal axle load limits. This affects pavement design more than geometric design, but the agent should note it as context — the design vehicle's legal weight may not represent the actual loading.

### M.3.3 Common Errors (Uganda-Specific)

1. **Assuming a specific international standard applies without checking the ToR.** The MoWT manual is the default. International standards (AASHTO, ERA) are only used when specifically required by the project Terms of Reference.
2. **Ignoring generated traffic on improvement projects.** Upgrading an unpaved district road to paved standard typically increases traffic 50–100% within 2–3 years. The post-improvement AADT should be used for classification.
3. **Not accounting for NMT.** In Uganda, pedestrians, cyclists, and motorcycles (boda-bodas) are a major share of road users on district and community roads. The classification should trigger NMT provision in the cross-section design (Skill #17, #20).

## M.5 Country-Specific Validation

### M.5.1 Range Checks

| Parameter | Minimum | Maximum | Unit | Standard Reference | Action if Violated |
|-----------|---------|---------|------|-------------------|-------------------|
| Design speed (MoWT) | 40 | 120 | km/h | MoWT Tables 4-3 to 4-9 | Warn — outside MoWT design class range |
| Design AADT for Design Class Ia | 12,000 | 20,000 | pcu/day | MoWT Table 4-2a | If outside range, review design class selection |

### M.5.2 Standards Compliance Checks

- **Check:** Design speed matches the tabulated value for the assigned design class and terrain.
- **Standard:** MoWT Tables 4-3 to 4-9
- **Pass condition:** Speed matches the value in the relevant table
- **Fail action:** Warn — "Design speed [X] km/h does not match Design Class [Y] on [terrain] terrain. Expected [Z] km/h."

- **Check:** Design class is appropriate for the functional class.
- **Standard:** MoWT Table 4-2a
- **Pass condition:** Design class is within the permitted range for the functional class (see Table U.2, "Applicable Functional Classes" column)
- **Fail action:** Warn — "Design Class [X] is not typically applicable to Functional Class [Y]. Review classification."

### M.5.3 Departures / Relaxations Process

Uganda MoWT does not have a formal departures process equivalent to DMRB. The manual states that "special conditions may demand modifications to these standards, in which case special consideration should be given in consultation with the Engineer-In-Chief" (MoWT Vol. 1, Section 1.1.3). Deviations are documented in the design report with engineering justification. For UNRA projects, non-standard designs require approval from UNRA's Engineering Department.

## M.7 Worked Example — Uganda

### Example 1: District Road in Eastern Uganda

**Source:** Hypothetical, based on typical Uganda MoWT practice.

**Given:**
- Road: Mbale–Bududa Road
- Country: Uganda
- Authority: Bududa District Local Government
- Function: Connects sub-county to district town — collector role
- Terrain: Rolling to mountainous (foothills of Mt. Elgon)
- Current AADT: 450 vpd (2024 manual count, 7-day adjusted)
- Growth rate: Not available
- Urban/rural: Rural with trading centres
- Funding: World Bank USMID-AF

**Solution:**

**Standard selection:** District road under local government → Uganda MoWT Road Design Manual (primary). World Bank funding noted but ToR does not specify a different standard.

**Terrain:** Rolling to mountainous. Predominant terrain: Rolling (60% of route). Use Rolling for classification; design mountainous sections at reduced speed.

**Design-year AADT:**
- Growth rate not available → Default 5% p.a. ⚠️
- Design period → 15 years ⚠️
- Design-year AADT = 450 × (1.05)^15 = 450 × 2.079 = **935 vpd (2039)**

**Road class:** MoWT, 935 vpd, collector function → **District Road Class B** (200–1,000 AADT). Functional check: Connects sub-county to district town = collector. ✅ Consistent.

**Design speed:** Class B, Rolling → **60 km/h**. Mountainous sections → **50 km/h**.

**Design vehicle:** SU (single-unit truck). ⚠️ Verify truck composition.

**Result:**

| Parameter | Value |
|-----------|-------|
| Standard | Uganda MoWT Road Design Manual 2010 |
| Road Class | District Road Class B |
| Design Speed | 60 km/h (rolling) / 50 km/h (mountainous) |
| Design Vehicle | SU |
| Design Period | 15 years |
| Design Year AADT | 935 vpd |

**Escalation:** 🔺 SENIOR REVIEW — Design-year AADT (935 vpd) is within 7% of the Class A/B boundary (1,000 vpd). At 6% growth the road classifies as District Road Class A with design speed 80 km/h. Recommend evaluating both classes with cost comparison.

### Example 2: National Trunk Road

**Source:** Hypothetical, based on typical UNRA practice.

**Given:**
- Road: Jinja–Kamuli Road (upgrade)
- Authority: UNRA
- Function: National trunk road (Functional Class B)
- Terrain: Flat to rolling
- Current AADT: 4,200 vpd (2023 ATC data)
- Growth rate: 6.5% (feasibility study)
- Design vehicle: WB-15 (ToR)
- Design period: 20 years (ToR)
- Funding: AfDB

**Solution:**

**Standard selection:** UNRA project → Uganda MoWT Road Design Manual (primary), with UNRA supplementary guidelines. AfDB funding — check ToR for additional requirements.

**Terrain:** Flat to rolling. Use Rolling as the governing terrain (conservative).

**Design-year AADT:** 4,200 × (1.065)^20 = **14,800 vpd (2043)**

**Functional class:** Class B (National Trunk — links provincial capitals and nationally important centres) ✅

**Design class:** 14,800 pcu/day, 4-lane required → **Design Class Ia Paved** (capacity 12,000–20,000 pcu/day)

**Design speed:** Design Class Ia, Rolling → **100 km/h** (from Table U.3)

**Result:**

| Parameter | Value |
|-----------|-------|
| Standard | Uganda MoWT Road Design Manual 2010 + UNRA supplementary guidelines |
| Functional Class | B (National Trunk) |
| Design Class | Ia Paved (dual carriageway) |
| Design Speed | 100 km/h |
| Design Vehicle | WB-15 |
| Design Period | 20 years |
| Design Year AADT | 14,800 vpd |

**Escalation:** None — all parameters within normal ranges for Design Class Ia.
